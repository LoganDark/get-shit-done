**Edit 3 of 4 — `sdk/src/vcs/parse/git-rev.ts`:**

Git has no "children" revset operator in the strict sense. The portable equivalent is `git log --reverse --ancestry-path <rev>..HEAD --format=%H -n 1`, but a `RevisionExpr` translation must yield a string that can be substituted into a `git rev-list`-style command without restructuring the call site.

Per RESEARCH §"`expr.children` factory addition" — the git-side mapping is documented as "planner confirms during plan 06-01 implementation." Confirmed strategy (chosen here):

Git's revision-syntax for direct children does not exist as a single token. The git backend will translate `children:<rev>` to a SENTINEL token `<rev>^!`  (which means "the commit `<rev>` and no parents" — NOT children, but a token that the git-side `vcs.log` consumer recognises and replaces with the multi-arg invocation `--reverse --ancestry-path <rev>..HEAD --not <rev>`).

However: **`vcs.log` is not called from git-side `orphan.ts` ancestor walks in plan 06-02** — the orphan walker only invokes `vcs.log({ rev: expr.children(...) })` when the ORPHAN is in jj's DAG (the "direct children in target VCS DAG" lookup). On the reverse direction (jj→git rewrites where the orphan is a jj change_id), the walker uses `expr.parents` (jj-side) for the walk and queries git-side for children only via `commitIdOf`.

So the git translator can throw `NotImplementedError` for the `children:` form, gated on a comment that documents the asymmetry. Plan 06-02's orphan walker is responsible for not calling `vcs.log` with `expr.children` on the git backend.

```typescript
  // 'children:<inner>' — git has no single-token children operator. Plan 06-02
  // restricts vcs.log({ rev: expr.children(...) }) calls to the JJ adapter
  // (the children-in-target-VCS lookup runs against jj only). If a future
  // caller needs git-side children, switch to git rev-list --ancestry-path.
  if (encoded.startsWith('children:')) {
    throw new Error(
      `parse/git-rev: 'children:' form is not supported on the git backend. ` +
      `git has no single-token direct-children revset operator. ` +
      `Use 'git rev-list --ancestry-path <rev>..HEAD --not <rev>' at the call site.`
    );
  }
```

This is the right tradeoff: a deterministic typed-error on misuse beats a silent-wrong translation. Plan 06-02 explicitly opts into the asymmetry.

Empirical probe for RESEARCH Assumption A5 (`x+` returns only direct children, not transitive descendants). Build a 3-level lineage (A → B → C), query A's children, assert ONLY B comes back (not C).

```typescript
import { describe, it, expect, afterAll, beforeAll } from 'vitest';
import { initJjColocated, rmTempDir } from './vcs-fixture.js';
import { writeFileSync } from 'node:fs';
import { execSync } from 'node:child_process';
import { join } from 'node:path';

function jjCmd(cwd: string, args: string): string {
  return execSync(
    `jj ${args} --no-pager --color never --quiet`,
    { cwd, encoding: 'utf8' }
  ).trim();
}

describe('Phase 6 probe A5: jj x+ operator returns ONLY direct children (depth-1)', () => {
  let dir: string;
  let changeA: string;
  let changeB: string;
  let changeC: string;

  beforeAll(() => {
    dir = initJjColocated('gsd-phase6-a5-');
    writeFileSync(join(dir, 'a.txt'), 'A\n');
    jjCmd(dir, `squash -B @ -k -m 'A'`);
    changeA = jjCmd(dir, `log -r '@-' -T 'change_id' --no-graph -n 1`);
    writeFileSync(join(dir, 'b.txt'), 'B\n');
    jjCmd(dir, `squash -B @ -k -m 'B'`);
    changeB = jjCmd(dir, `log -r '@-' -T 'change_id' --no-graph -n 1`);
    writeFileSync(join(dir, 'c.txt'), 'C\n');
    jjCmd(dir, `squash -B @ -k -m 'C'`);
    changeC = jjCmd(dir, `log -r '@-' -T 'change_id' --no-graph -n 1`);
  });

  afterAll(() => rmTempDir(dir));

  it('children of A returns ONLY B, not C', () => {
    const out = jjCmd(dir, `log -r '${changeA}+' -T 'change_id ++ "\\n"' --no-graph`);
    const ids = out.split(/\n+/).filter(s => s.length > 0);
    expect(ids).toContain(changeB);
    expect(ids).not.toContain(changeC);
    expect(ids).not.toContain(changeA);   // x+ does not include x itself
  });

  it('children of B returns ONLY C', () => {
    const out = jjCmd(dir, `log -r '${changeB}+' -T 'change_id ++ "\\n"' --no-graph`);
    const ids = out.split(/\n+/).filter(s => s.length > 0);
    expect(ids).toContain(changeC);
    expect(ids).not.toContain(changeA);
    expect(ids).not.toContain(changeB);
  });

  it('descendants (x::) DOES include transitive children — establishes that x+ is strictly stricter', () => {
    const out = jjCmd(dir, `log -r '${changeA}::' -T 'change_id ++ "\\n"' --no-graph`);
    const ids = out.split(/\n+/).filter(s => s.length > 0);
    expect(ids).toContain(changeA);   // x:: includes x
    expect(ids).toContain(changeB);
    expect(ids).toContain(changeC);
  });
});
```
