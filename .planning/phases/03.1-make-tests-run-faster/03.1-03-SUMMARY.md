## Why skipped

`isolate: false` is a sharper trade-off than L1: it skips vitest's per-test-file module reset, so any test that mutates module-scope state (e.g., singletons, env stubs, mocks) becomes order-dependent. RESEARCH already established this risk as LOW for the 7 integration files directly but MEDIUM for transitive imports, which is why the plan required a pre-flip audit (`grep -r 'vi.mock' …` across the import closure of each integration file) before applying L2.

Given:
- Plan 02 skipped (same correctness/perf math applies to L2 at 7.4 s).
- The audit itself would take a measurable fraction of the time saved.
- The user's signal after running `pnpm vitest` directly was "acceptable performance now."

…there's no reason to flip L2. The 22-LOC vitest config seed remains untouched.
