    **Step 2 — Classify findings:**
    For each hit (or `CLEAR`), classify into one of three buckets:
    - **CLEAR** — no hits, no action needed.
    - **BENIGN** — hits exist but are safe under L2. Examples:
      - Top-level `const cache = new Map()` where the cache is content-addressed (e.g., file-path → result) and not mutated by test setup → safe (worker reuse just keeps the cache warm).
      - `process.on('exit', ...)` registered to log/cleanup → benign because worker threads don't run process exit handlers anyway, but document.
      - Top-level `let _foo` that is written once via try/catch at import time and never re-mutated → safe (same value across files in the same worker).
    - **HAZARD** — hits that would corrupt cross-file behavior:
      - `vi.mock(...)` calls (mock persists across files; first file's mock affects subsequent files)
      - `vi.stubEnv(...)` without a paired `vi.unstubAllEnvs()` in afterAll
      - Top-level `let counter = 0; counter++` style state that test setup mutates
      - Module-level Map/Set that test setup writes into (singleton registry mutated by tests)
      - Signal handlers registered conditionally on env (race under worker threads)
