## Decisions Made

1. **One atomic commit, not split per file.** PLAN-06 task 2 prescribes `Stage the 4 files (worktree-safety.cjs + 3 test files); … commit.` Per D-21, the dimension's dedicated commit is a single `refactor(2.1-06): …` landing all consumer sites together. Splitting would have created a transiently-broken type state on intermediate commits.

2. **Retained `vcs.workspace.context()` invocation in resolveWorktreeContext** even though its return value (ctx) is no longer needed for gitDir/gitCommonDir. The throw-on-non-repo behavior is the surviving not_git_repo gate; replacing it with a bare existence check would be a semantics shift outside D-18 scope.

3. **adapter-contract.test.ts probe wrapped in `vcs.kind === 'git'`** rather than dropped. The test is the canonical cross-backend assertion that workspace.context returns the same mode/isLinked shape on both backends; keeping the gitOnly probe inside a narrow preserves the original intent (assert main-repo equivalence) without breaking the jj-backend invocation path.

4. **git-backend.test.ts main-repo test retains the gitDir === gitCommonDir assertion (now via gitOnly)** even though it is tautological under the post-D-18 shape. It still asserts that on a non-linked worktree both gitOnly methods resolve to the same path — a meaningful contract test of the gitOnly surface.
