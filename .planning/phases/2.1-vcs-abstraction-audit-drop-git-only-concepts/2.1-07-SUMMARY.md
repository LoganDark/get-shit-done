## Decisions Made

1. **DELETE every site, do not reframe.** RESEARCH Open Q1 explicitly defers internal-invocation rewiring to Phase 4 (HOOK-01..05). A reframed test in 2.1 would either need to call the still-private `fireHook` directly (which would require re-exporting it, contradicting plan 01) or assert side-effects from a hook script (which depends on internal wiring not yet present in `commit()` / `push()`). Both routes pre-empt Phase 4 design. Comment breadcrumbs preserve the audit trail without keeping a broken test.

2. **Hooks describe block in `git-backend.test.ts` deleted in full** (lines 209-215 pre-edit), not just the inner `vcs.hooks.fire` line. The block contained exactly one it() with the broken assertion; nothing else inside the describe survives D-07. Replacing it with an empty describe block would be noise.

3. **Frozen-depth describe blocks preserved with the `vcs.hooks` assertion removed.** Two scopes (`git-backend.test.ts` line 629-639, `index.test.ts` lines 33-43, `vcs-adapter-contract.test.cjs` lines 59-68) probe `Object.isFrozen` across the adapter shape. Each retains its meaningful assertions (vcs, refs, refs.bookmarks, workspace, gitOnly) — only the `vcs.hooks` line is dropped. `Object.isFrozen(vcs.hooks)` would be `Object.isFrozen(undefined) === false` post-plan-01 (namespace no longer exists on the adapter), so removal both fixes the test and reflects the new shape correctly.

4. **`index.test.ts` describe-text label updated** from `'refs/refs.bookmarks/workspace/hooks/gitOnly'` to `'refs/refs.bookmarks/workspace/gitOnly'` so the it() description string reflects what the test actually probes post-D-07. Mechanical, no behavior change.

5. **No re-export of `fireHook` from `hook-bridge.ts`.** Plan 01 made it module-private; this plan does not undo that. Any Phase 4 access path will go through an internal call in the same module or a new internal seam — re-exposing it publicly would re-create the deletion target.
