### Minor notes (NOT deviations)

1. **baseline-parity.test.ts rev-parse --abbrev-ref dispatch needed an empty-stdout discriminator.** The PLAN explicitly anticipated this: "If the baseline JSON's expected.stdout is 'feat-cb\n', the trimmed canonical is 'feat-cb' and the adapter returns ['feat-cb']" — the implementation routes empty stdout and the literal `'HEAD'` (canonical detached-HEAD stdout) to `toEqual([])` while non-empty stdout routes to `toEqual([stdout])`. This is one structural addition (an `if`/`else` inside the existing dispatch clause), no logic change to the dispatcher itself.

2. **git-backend.test.ts describe block heading text updated from "(02-03 Task 1)" to "(2.1-03)".** This is a cosmetic correction — the test block now correctly attributes the currentBookmarks shape to plan 2.1-03 rather than the original currentBranch-introducing plan. Not a behavior change.

3. **adapter-contract.test.ts test name + body shape changed beyond a 1:1 rename.** The original test asserted `typeof cb === 'string'` + length>0 (a single-string-shape probe). The reshaped test asserts `Array.isArray(cb)`, `cb.length > 0`, then probes `typeof cb[0]` + `cb[0]!.length > 0`. This is the consumer-correct symmetric-contract analog: the original test's intent (the backend reports a non-empty current-branch identifier on a fresh init) is preserved against the new shape.
